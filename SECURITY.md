# Politica de seguranca

## Uso autorizado

Esta ferramenta deve ser usada apenas em computadores proprios ou em ambientes onde o tecnico tenha autorizacao explicita para realizar manutencao.

## Dados sensiveis

Antes de abrir uma issue, compartilhar logs ou enviar relatorios, remova informacoes como:

- nomes de usuarios;
- nomes de maquinas;
- enderecos IP internos;
- caminhos de rede;
- dados de empresas ou clientes;
- qualquer informacao de autenticacao.

## Relato de problemas

Caso encontre comportamento perigoso, comando incorreto ou risco de exposicao de dados, abra uma issue no repositorio descrevendo:

- versao do Windows;
- versao do Python;
- opcao executada;
- resultado esperado;
- resultado obtido;
- mensagem de erro sem dados sensiveis.
